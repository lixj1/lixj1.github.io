## 理解摘要

- 当前项目的“AI 协作经验库”已经有 3 条方法内容，现需补充第 4 条。
- 新增内容聚焦 Speckit 的功能介绍与安装，不涉及新增页面或导航结构调整。
- 重点信息包括：基于 SSD 开发框架、clarify 用于需求细化提效、feature 结果沉淀在 /specs 方便 code review。

- 当前项目是一个单文件静态文档站，主体在 [index.html](index.html)。
- 当前页面已经有“Git 方法”栏目，并已接入 [git方法.mp4](git方法.mp4)。
- 你现在进一步要求做两项增强：补封面图，以及补关键命令摘要。
- 这两个增强都只针对 Git 视频页，不影响其他栏目结构。

## 假设

- 假设 Speckit 条目沿用现有“问题是什么 / 关键思路 / 适用场景 / 如何安装 / 具体怎么做 / 为什么有效”的表达节奏。
- 假设安装说明以官方 quickstart 中的 `uvx --from git+https://github.com/github/spec-kit.git specify init .` 为主，Windows 场景补充 `--script ps` 可选写法。

- 假设仍然只维护 [index.html](index.html) 一个交付文件，不新增框架和构建流程。
- 假设封面图采用本地静态文件形式，和视频同级存放即可。
- 假设关键命令摘要以页面内静态说明卡片呈现，不需要交互展开。
- 假设你希望保留当前视觉风格，不做大规模 UI 重做，只做信息架构上的增删。

## 决策记录
