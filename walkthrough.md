## 架构说明

这轮之后，站点顶层信息架构正式分成四类：

1. 首页：环境配置与模型选型
2. Git 方法：Git 存档视频与留档操作
3. AI 协作经验库：团队内部已经验证过的方法与工作流
4. AI Prompt 模板库：可直接复用的 Prompt 模板

最新命名下，这两块内容的边界已经更清楚：

1. AI 协作经验库：方法、节奏、协作套路
2. AI Prompt 模板库：可以直接复制和改写的 Prompt 资产

随着 Spec Kit 被补充进“AI 协作经验库”，这一栏目前已经覆盖四类典型协作方式：项目级角色固化、长 Session 转场、工程级质量约束，以及基于 SSD 的规格驱动协作。这样团队在面对不同复杂度的问题时，可以更快判断该先“改 Prompt”、还是该先“改流程”。

把“AI 协作经验库”单独提出来，而不是继续塞进“AI Prompt 模板库”，是因为两类内容的阅读目的完全不同：Prompt 是拿来直接复制用的，方法文章是拿来建立协作共识的。如果混在一起，后面会越来越难找，也会让栏目边界变得模糊。

这次没有继续把“项目文件清理”做成独立顶层页面，而是归并到“AI Prompt 模板库”页内，原因有两个：

1. 你已经明确需要一个单独的 Prompt 资产页，后面还会不断增加很多提示词相关内容。
2. 如果每个 Prompt 都直接升为顶层导航，后续页面会快速失控；先做顶层导航，再做页内子导航，更适合知识库扩展。

因此当前结构是：

1. 首页：顶部引导 + 团队环境配置 + 模型选型 + 一句话结论
2. Git 方法：当前视频为“git方法.mp4”
3. AI 协作经验库：当前首篇为“AI 协作转场法”
4. AI Prompt 模板库：Prompt 资产沉淀页
5. 第一层级子项：受试者/参与者列名兼容改造
6. 第一层级子项：项目文件清理

## 为什么这么做

原文件更像一张漂亮的长海报，但不适合后续迭代。你现在的目标已经不是单纯展示，而是做团队长期使用的内部经验站，所以核心不是“把一页做满”，而是“让站点能继续长出来”。

这次改造的重点是先把骨架搭对：

1. 左侧固定导航，建立站点感。
2. 提示词页内再做子层级，给未来扩展留位置。
3. 用单文件交付，保持部署简单，方便直接挂到 GitHub Pages。

这轮扩展的重点是把“结构可扩展”真正用起来：

1. 用多个专题验证提示词页的分层模式是否成立。
2. 让子层级不仅能展示，还能通过 hash 定位和切换，后续继续加专题时不需要重写整套逻辑。

这一轮继续补专题的意义也很明确：

1. 让提示词页只保留当前真正需要的内容，减少无关专题干扰。
2. 用更贴近当前改造任务的 Prompt 替换泛化的“旧项目接手”专题，提高直接可用性。
3. 把当前最高频、最具体的专项提示词放到第一位，降低团队成员进入页面后的选择成本。
